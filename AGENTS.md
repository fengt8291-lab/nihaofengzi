# AGENTS.md

## 项目名称

你好丰子个人数字主页（nihaofengzi.top）


# 一、项目目标

本项目目标是构建一个长期维护的个人数字主页。

网站不是传统简历页面，而是一个个人品牌入口，用于展示：

- 个人介绍
- AI产品项目
- 学术研究
- 技术探索
- 作品集
- 联系方式


最终网站地址：

https://nihaofengzi.top


# 二、开发原则

开发过程中遵循以下原则：

1. 优先考虑长期维护性，而不是快速堆叠功能。

2. 保持代码结构清晰，组件划分合理。

3. 避免没有必要的复杂依赖。

4. 所有设计和技术选择需要考虑未来扩展。

5. 优先保证：

- 性能
- 响应速度
- 移动端体验
- SEO友好


# 三、技术方向

默认技术栈：

- Astro
- TypeScript
- Tailwind CSS
- React组件
- Framer Motion动画


如果引入新的技术或依赖，需要说明：

- 为什么需要
- 解决什么问题
- 是否影响长期维护


# 四、设计规范

整体视觉方向：

苹果官网风格。


关键词：

- 极简
- 高级
- 留白
- 现代
- 克制
- 注重体验


避免：

- 普通程序员博客风格
- 过度花哨动画
- 大量模板化组件
- 廉价商业网站视觉


动画要求：

动画应该服务于体验：

允许：

- 页面渐入
- 平滑滚动
- 卡片hover效果
- 微交互


避免：

- 影响加载速度的复杂动画
- 为炫技而存在的效果


# 五、开发流程要求

在进行较大修改之前：

先说明：

1. 修改目的
2. 实现方案
3. 影响范围


涉及架构调整时：

必须先进入计划模式（Plan Mode），输出方案后等待确认。


# 六、代码修改要求

每次修改后：

需要说明：

1. 修改了哪些文件
2. 每个文件的作用
3. 如何测试
4. 是否影响部署

# 八、内容原则

网站内容应该突出：

- AI产品探索
- 数字化管理学习背景
- 人机协作研究
- 独立开发实践


网站定位：

展示一个持续探索AI时代个人成长方式的年轻实践者。


不要包装成：

- 已经成功的企业家
- 虚假的专家身份


所有内容应该：

真实
克制
有思考
具有个人特色。
